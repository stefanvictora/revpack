# Review Bundles Use Per-File Anchor Maps

Revpack separates diff understanding from positional anchoring: standard per-file patches remain the artifact for understanding changes, while `files.json` indexes one NDJSON Anchor Map per changed file for selecting valid positional anchors from the full current diff. Each Anchor Map preserves diff order and contains only the line kind, applicable old and new line numbers, and text; paths and file identity stay in `files.json`, absent line fields are omitted, and files without valid anchors receive an empty map. Hunk boundaries remain an internal patch-parsing concern, while public hunk metadata, the global `line-map.ndjson`, and `change-blocks.json` are removed because they duplicate patch structure and enlarge the agent-facing interface; publishing continues to validate findings independently by reparsing `latest.patch`.
